Address Resolution Protocol

Contributors:

  • none yet

ARP is used as a method for dynamically discovering the mapping between an IP address and a physical Ethernet address.  If you want to send a packet to some arbitrary IP address (32-bit), the network drivers first have to find out what physical address (48-bit) to send it to.  So it asks ARP, which first checks a cache of the mappings it already has.  If it's not there, then it has to transmit a broadcast packet to ask the machine on the network with that IP address what it's real address is.  If it's asking for an IP address not on the local net then it should get the gateway's physical address.
